CMAM Documentation

Connor Merk App Manager — A lightweight package manager for installing and managing Connor Merk's applications on Windows.

v2.4.1  Windows  MIT License

Simple Installation

One-command install for any supported application from the CMAM repository.

Automatic PATH

Installed apps are automatically added to your system PATH. Just restart your terminal.

Version Control

Install specific versions, update, rollback, and keep backups of previous releases.

Checksum Verification

SHA256 checksums ensure every downloaded binary is authentic and untampered.

Self-Updating

Keep CMAM itself up to date with a single self-update command.

Export & Import

Export your app list to a manifest and re-import on a fresh machine in seconds.

Installation

CMAM can be installed via the automated installer or manually. It requires Windows and installs to C:\.cmam.

Quick Install (Recommended)

Run the following in PowerShell:

PowerShell 
# Download the installer
Invoke-WebRequest -Uri "https://github.com/cmerk2021/cmam/releases/latest/download/install.exe" -OutFile "install.exe"

# Run the installer
.\install.exe

Alternatively, download and run install.exe directly from the latest release or the /dist directory.

Installer Options

OptionDescription
--version, -vInstall a specific version of CMAM
--skip-pathSkip adding CMAM to the system PATH
--force, -fForce reinstall even if CMAM is already installed

Manual Installation

Download the binary

Download cmam.exe from the latest release on GitHub.

Create the directory

Create the folder C:\.cmam\scripts if it doesn't already exist.

Place the executable

Move cmam.exe into C:\.cmam\scripts.

Add to PATH

Add C:\.cmam\scripts to your User PATH environment variable.

Restart your terminal

Open a new terminal window and verify with cmam --version.

Updating CMAM

Once installed, update CMAM by running:

Terminal 
cmam self-update
Auto Update Checks

CMAM automatically checks for new versions every time you run a command. If a newer version is available, you'll see a notification prompting you to run cmam self-update.

Directory Structure

CMAM installs and manages everything under the C:\.cmam root directory:

File System 
C:\.cmam\
├── scripts\          # Installed app executables + CMAM itself
│   ├── cmam.exe      # CMAM executable
│   ├── myapp.exe     # Example installed app
│   └── helper.dll    # Example dependency file
├── .cache\           # Temporary download cache
│   └── backups\      # Versioned backups of updated apps
├── packages.json     # Installed packages metadata
└── packages.txt      # Package list (legacy)
PathPurpose
C:\.cmam\scripts\All installed executables and dependency files live here. This folder is added to your PATH.
C:\.cmam\.cache\Temporary files during downloads. Safely cleaned with cmam clean.
C:\.cmam\.cache\backups\Versioned backups created during updates (e.g., myapp_v1.0.0.exe.bak).
C:\.cmam\packages.jsonJSON metadata tracking every installed app, its version, and its dependency files.

CLI Reference

Complete reference for all CMAM commands. Run cmam --help or cmam <command> --help for built-in usage info.

Global Options
OptionDescription
--version, -VShow CMAM version and exit
--helpShow help message and exit

cmam install

Installs a specified application from the CMAM repository. Downloads the latest release (or a specific version), verifies the SHA256 checksum, places the executable in the scripts folder, handles dependency files, and ensures the PATH is configured.

Terminal 
# Install the latest version
cmam install <app-name>

# Install a specific version
cmam install <app-name> --version 1.2.0
Argument / OptionTypeRequiredDescription
app_namestringYesName of the application to install
--version, -vstringNoSpecific version to install (e.g., 1.2.0)
Tip

If a release contains multiple .exe assets, the primary one is renamed to <app-name>.exe and additional executables keep their original filenames. Dependency files listed in the manifest are also downloaded automatically.

cmam update

Updates an installed application to the latest version or a specified version. Automatically creates a backup of the current version before updating.

Terminal 
# Update to the latest version
cmam update <app-name>

# Update to a specific version
cmam update <app-name> --version 2.0.0

# Update without keeping old backup
cmam update <app-name> --no-keep-backup
Argument / OptionTypeDefaultDescription
app_namestring—Name of the application to update (required)
--version, -vstringlatestSpecific version to update to
--keep-backup, -kflagtrueKeep a versioned backup of the old binary

cmam update-all

Checks all installed applications for available updates and bulk-updates them. Shows a summary table before proceeding and asks for confirmation.

Terminal 
# Check and update all apps (with confirmation prompt)
cmam update-all

# Skip confirmation
cmam update-all --yes

# Update all without keeping backups
cmam update-all --no-keep-backup
OptionTypeDefaultDescription
--yes, -yflagfalseSkip the confirmation prompt
--keep-backup, -kflagtrueKeep backups of old binaries

cmam uninstall

Removes an installed application, its dependency files, and optionally its backups. Shared dependencies (used by other installed apps) are preserved.

Terminal 
# Uninstall an app (removes backups too)
cmam uninstall <app-name>

# Uninstall but keep backup files
cmam uninstall <app-name> --keep-backups
Argument / OptionTypeDefaultDescription
app_namestring—Name of the application to uninstall (required)
--keep-backups, -kflagfalseKeep backup files for this app
Shared Dependencies

If a dependency file is shared with another installed app, it won't be removed. CMAM tracks which dependencies belong to which packages in packages.json.

cmam list

Displays a table of all installed applications, their versions, number of backups, and component files (dependencies).

Terminal 
cmam list

Example output columns: App Name, Version, Backups, Components.

cmam search

Searches the remote CMAM repository for applications matching a query. The search is case-insensitive and matches against app names.

Terminal 
cmam search <query>
ArgumentTypeRequiredDescription
querystringYesSearch query to match against app names

cmam info

Shows detailed information about an application — repository link, description, latest version, installed version, update availability, backups, and component files. If you query a dependency or secondary executable, CMAM automatically redirects to the parent package.

Terminal 
cmam info <app-name>

Verify & Validate

CMAM provides two commands for integrity checking of installed applications.

cmam verify

Verifies the SHA256 checksum of a single installed app against the official release. Reports whether the binary is authentic or potentially corrupted.

Terminal 
cmam verify <app-name>

cmam validate

Validates all installed apps at once. Produces a table showing each app's verification status: Valid, Missing, Checksum Mismatch, or Unknown.

Terminal 
cmam validate

Repair & Rollback

cmam repair

Repairs a corrupted app by removing the existing executable and reinstalling the same version from the remote repository.

Terminal 
cmam repair <app-name>

cmam rollback

Rolls back an app to a previously backed-up version. Without --version, it lists available backup versions. With --version, it restores the specified backup.

Terminal 
# List available backups
cmam rollback <app-name>

# Rollback to a specific version
cmam rollback <app-name> --version 1.0.0
Argument / OptionTypeRequiredDescription
app_namestringYesName of the application to roll back
--version, -vstringNoSpecific backup version to restore. Omit to list available backups.
Backups Required

Rollback only works if backups exist. Backups are created automatically during cmam update (when --keep-backup is enabled, which is the default). If you ran cmam clean --backups, old backups will be gone.

Export & Import

Move your CMAM environment between machines or share your setup with others.

cmam export

Exports your list of installed apps and their versions to a JSON manifest file.

Terminal 
# Export to default file (cmam-export.json)
cmam export

# Export to a custom file
cmam export --output my-apps.json

The exported JSON has the following structure:

JSON 
{
  "cmam_version": "2.4.1",
  "apps": {
    "myapp": { "version": "1.3.0" },
    "otherapp": { "version": "2.0.1" }
  }
}

cmam import

Installs all apps listed in a manifest file. Skips already-installed apps by default.

Terminal 
# Import from a manifest file
cmam import cmam-export.json

# Import and reinstall even if already present
cmam import cmam-export.json --no-skip-existing
Argument / OptionTypeDefaultDescription
input_filestring—Path to the manifest JSON file (required)
--skip-existing, -sflagtrueSkip apps that are already installed

Self-Update & Self-Repair

cmam self-update

Checks for a newer version of CMAM and updates itself. Creates a backup of the current CMAM binary, downloads the new version, and uses a batch script to replace the executable after the process exits.

Terminal 
cmam self-update

cmam self-repair

Repairs the CMAM installation by downloading and running the latest installer with -f (force). Cleans up old CMAM cache, backups, and temp files. Does not affect installed applications.

Terminal 
cmam self-repair
When to use self-repair

Use self-repair if self-update fails, if your CMAM binary is corrupted, or if the installation directories are in a broken state. It re-runs the full installer while preserving your installed apps.

Utility Commands

cmam doctor

Runs a comprehensive health diagnostic on your CMAM installation. Checks:

  • Required directories exist (root, cache, scripts, backups)
  • packages.json is valid and not corrupted
  • Scripts folder is in your PATH
  • All registered apps have matching executables
  • No orphaned executables in the scripts folder
  • GitHub API is reachable
  • Remote manifest is accessible
  • Adequate disk space available
Terminal 
cmam doctor

cmam clean

Cleans up cache files, temporary downloads, and optionally orphaned apps and backups.

Terminal 
# Clean cache and orphans (default)
cmam clean

# Also delete all backups (use with caution!)
cmam clean --backups

# Only clean cache, skip orphan removal
cmam clean --no-orphans
OptionTypeDefaultDescription
--cache / --no-cacheflagtrueClean cache/temp files
--orphans / --no-orphansflagtrueRemove orphaned executables not tracked in packages.json
--backups, -bflagfalseAlso remove all backup files (asks for confirmation)
Danger

Using --backups permanently deletes all versioned backups. You will not be able to use cmam rollback afterward. This action cannot be undone.

cmam trust

Displays the trust/verification status of all installed applications by comparing local SHA256 checksums against the official releases. Shows a table with statuses: Trusted, Untrusted, Unknown, or Missing.

Terminal 
cmam trust

cmam path

Checks whether the CMAM scripts folder (C:\.cmam\scripts) is in your User PATH. Optionally re-adds it.

Terminal 
# Check PATH status
cmam path

# Add to PATH if missing
cmam path --add
OptionTypeDefaultDescription
--add, -aflagfalseAdd the scripts folder to PATH if not present

Typical Workflows

Fresh Setup

Install CMAM

Download and run the installer, or use the PowerShell one-liner above.

Search for apps

Run cmam search <keyword> to find available applications.

Install your apps

Run cmam install <app-name> for each application you need.

Export your setup

Run cmam export to save your app list. Keep the manifest for future migrations.

Migrating to a New Machine

Export from old machine

Run cmam export --output my-setup.json and copy the file to the new machine.

Install CMAM on new machine

Install CMAM via the normal installer.

Import your apps

Run cmam import my-setup.json to install all your apps at once.

Troubleshooting

CMAM is not recognized as a command

Make sure C:\.cmam\scripts is in your User PATH. You can check and fix this by running cmam path --add from a working installation, or manually add the directory in Windows Settings > Environment Variables. Always restart your terminal after PATH changes.

Download fails or checksum mismatch

Check your internet connection first. The GitHub API may also be temporarily unavailable. Try again in a few minutes. If the problem persists, run cmam doctor to check network connectivity and manifest accessibility.

Permission errors

Some operations (especially PATH modifications) may require administrator privileges. Try running your terminal as Administrator. The cmam path --add command modifies the HKEY_CURRENT_USER\Environment registry key.

App binary appears corrupted

Run cmam verify <app-name> to check the file's integrity. If the checksum doesn't match, run cmam repair <app-name> to re-download and reinstall the same version.

An update broke my app

If backups are available, run cmam rollback <app-name> to see backup versions, then cmam rollback <app-name> --version X.Y.Z to restore. Backups are created automatically during updates unless you used --no-keep-backup.

CMAM itself is broken

If cmam self-update fails, try cmam self-repair. This downloads the latest installer and force-reinstalls CMAM while preserving all your installed applications. As a last resort, do a manual installation.

Version Scheme

CMAM uses Semantic Versioning (MAJOR.MINOR.PATCH) with optional pre-release tags. The version parser supports:

FormatExampleSort Priority
Stable release2.4.1Highest
Release candidate2.5.0-rcHigh
Beta2.5.0-betaMedium
Alpha2.5.0-alphaLowest

Version tags in GitHub releases should be prefixed with v (e.g., v2.4.1). CMAM strips the v prefix automatically.
